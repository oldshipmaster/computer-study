# 比特岛探险任务循环设计

## 目标

在不增加课程时长、不制造每日签到压力、也不引入付费或抽奖的前提下，让现有 65 节课程更值得主动选择和重复游玩。孩子每次回到地图都能看到一个短小、明确、可完成的探险目标，并通过真实学习行为获得持续可见的成长反馈。

## 设计原则

- 面向 7 岁小学二年级孩子，文字短、按钮直接、反馈具体。
- 一次仍只推荐一节 8–10 分钟课程，不鼓励连续长时间使用屏幕。
- 奖励学习行为，不奖励在线时长、连续天数或反复点击。
- 不使用随机付费、倒计时、稀有概率和“错过即失去”等压力机制。
- 所有记录只保存在当前浏览器，并进入现有导出、恢复、重置流程。
- 任务由既有课程与本机进度生成，不向网络发送学习记录。

## 已比较的路线

### 路线 A：签到与连续学习

优点是反馈直接，实现简单。缺点是会把“每天都要来”放在学习本身之前，也容易让孩子为了保持数字而延长屏幕时间，因此不采用。

### 路线 B：探险任务牌与能量等级（采用）

把现有课程重新组合成三种有目的的任务，并让首次完成与有效重玩都积累能量。它能提升课程发现、复习和薄弱点修补，同时不依赖日期。

### 路线 C：比比换装商店

收藏感较强，但如果先做完整商店，学习目标会被货币和物品列表抢走注意力。本轮只做少量自动解锁的装备等级，保留未来扩展空间。

## 核心玩法

地图首屏在“五次探险计划”之前增加“比比的探险任务牌”。任务牌每轮最多三张，课程不重复：

1. **开新航线**：推荐尚未完成的下一节课程。完成后获得首次完成能量。
2. **回访藏宝洞**：从已完成课程中挑选游玩次数较少的一课，鼓励真正的间隔重玩。
3. **修补能量站**：优先选择孩子标记为“我想再练”或“请大人帮忙”的课程；没有薄弱课程时，选择一节已完成但重玩较少的课程，并提示用自己的话讲出原理。

未完成任何课程时只显示“开新航线”，避免用灰色任务制造挫败感。已经完成全部 65 课时，“开新航线”自动变为“随机远征”，从低频课程中选择一课。

任务生成必须确定且可测试：同一份进度和同一个轮换编号产生相同任务。孩子可以点击“换一批任务”，只改变本机组件中的轮换编号，不消耗能量，也没有次数限制。

## 探险能量与等级

课程真正完成时更新该课程的游玩次数：

- 每门课程首次完成：10 点能量。
- 同一课程第 2、3 次完成：每次 3 点能量。
- 第 4 次及以后仍可自由重玩，但不再增加能量，避免机械刷分。

能量完全由游玩次数推导，不单独存储余额，防止记录不一致。地图显示当前能量、距离下一等级还差多少，以及一条可访问的进度条。

装备等级自动解锁，不需要购买：

- 0 点：启航护目镜
- 60 点：星图天线
- 180 点：数据披风
- 360 点：量子背包
- 650 点：群岛探险冠

装备以 CSS 形状和文字标识呈现，沿用现有比比视觉，不增加大图片下载。解锁只改变外观称号和庆祝反馈，不改变课程难度。

## 数据与兼容性

现有 `bit-island-progress-v1` 增加可选字段 `coursePlayCounts`，值为课程 ID 到 1–3 的映射。解析时只接受长度受限的字符串键和 1–3 的整数；旧记录没有该字段时，根据已完成课程补为 1。这样旧用户不会失去首次完成奖励，也不会重复获得它。

`completeCourse` 每次真正领取课程完成奖励时递增对应次数，并封顶为 3。现有课程 ID、徽章和完成列表继续去重。

备份文件继续保持版本 1，并把经过白名单处理的 `coursePlayCounts` 纳入导出和恢复。恢复与启动时只保留当前 65 课中的已知课程；未完成课程不能保留游玩次数。重置进度会清空次数但保留声音与减少动画设置。

## 组件边界

- `lib/adventure-missions.ts`：纯函数，负责任务挑选、能量计算、等级计算和下一等级进度。
- `components/AdventureMissionBoard.tsx`：渲染任务牌、换一批按钮、能量仪表与装备展示；只接收进度和开始课程回调。
- `components/IslandMap.tsx`：把进度传给任务牌，并保持现有地图与课程筛选职责不变。
- `components/BitIslandApp.tsx`：继续作为进度唯一拥有者，把游玩次数传入地图。
- `lib/progress.mjs`、备份与目录清洗模块：负责新字段的可信边界。

任务牌不直接修改进度；只有课程完成事件可以增加游玩次数。这样点击“开始”或中途退出不会获得奖励。

## 交互与可访问性

- 所有任务都是真实 `button`，支持键盘、触摸和鼠标。
- “换一批”有明确的状态提示，读屏可知道任务已更新。
- 能量进度使用原生 `progress` 并提供文本等价信息。
- 减少动画设置开启时，翻牌和装备庆祝不做位移或旋转。
- 任务卡至少 44px 触控高度，小屏单列排列。
- 不自动播放声音；继续尊重现有声音设置。

## 错误与边界处理

- 无已完成课程：仅显示开新航线。
- 全部完成：三张任务都从低频重玩课程中生成，并保持课程不重复。
- 所有课程都重玩三次：仍可生成任务，但文案改为“自由巩固”，不承诺额外能量。
- 存储不可用：任务仍依据本次内存进度工作；沿用现有存储提示。
- 恶意或损坏记录：解析、目录清洗和备份恢复三层都限制已知课程、整数范围和条目数量。

## 测试与验收

1. 先为进度解析、首次完成、两次有效重玩、封顶、旧记录迁移和重置写失败测试。
2. 为任务生成写纯函数测试，覆盖新用户、混合进度、薄弱点、全完成、确定性、轮换和不重复。
3. 为能量等级写边界测试，覆盖每个阈值和满级状态。
4. 为组件源码与静态渲染补充结构、按钮、进度条、触控和减少动画检查。
5. 运行完整生产构建、全部测试、类型检查、代码检查、依赖审计和 GitHub Pages 静态输出测试。
6. 发布后核对 GitHub Actions 成功、公开页面 HTTP 200，并确认新任务牌文本和资源已上线。

## 本轮不做

- 每日签到、连续天数、排行榜和社交比较。
- 付费、广告、抽奖、概率掉落或可消耗货币。
- 账号、云同步和跨设备身份。
- 大型换装商店、自由文本昵称或照片上传。
- 改写 65 节课程内部实验；本轮集中增强跨课程游戏循环。
