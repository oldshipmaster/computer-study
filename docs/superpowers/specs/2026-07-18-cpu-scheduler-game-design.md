# CPU 时间片调度台设计

## 目标

把“程序变进程、CPU 轮转调度、内存申请与释放”组合成一个可操作的资源管理游戏。孩子在有限内存中选择装入任务，再让 CPU 每次只执行一个时间片，观察就绪队列轮转、剩余工作减少和完成任务释放内存。

## 解锁与位置

- 完成 `program-process`、`cpu-scheduling`、`memory-allocation` 后解锁。
- 位于网络数据包护航之后、五次学习计划之前。
- 未解锁时显示 0–3 门进度和下一课入口。
- 组件及样式独立懒加载。

## 六个调度班次

- 每班包含 2–5 个虚构任务、4–8 格内存容量和 1–4 个时间片的工作量。
- 一次装入任务会申请对应内存；容量不足时拒绝并保留现场。
- 点击“运行一个时间片”时，只执行就绪队首任务。
- 未完成任务移动到队尾，形成轮转；完成任务离开队列并自动释放内存。
- 内存释放后，孩子可继续装入等待任务，直到全部完成。
- 六班完成后可轮换重玩。

## 反馈与证据

- 内存用格子和文字同时显示占用者、已用和剩余。
- 就绪队列明确标出队首、队尾；CPU 显示刚执行的任务。
- 时间片历史保留最近若干步，说明任务执行、轮转或完成释放。
- 容量不足不丢任务，不回退已完成工作，不给失败惩罚。
- 所有动作由纯函数引擎限制阶段、ID、容量和重复点击。

## 儿童体验与隐私

- 无倒计时、生命值、排行榜、随机奖励或真实系统 API。
- 所有任务均为虚构，不读取电脑中的真实程序、内存或进程。
- 按钮至少 44px，支持键盘、屏幕阅读器、移动端、减少动态和强制颜色。
- 每班约 1–2 分钟，六班约 8–10 分钟。
- 队列、内存和历史仅在当前页面内存，不写入学习记录或备份。

## 验证

- 引擎测试覆盖任务数据、容量上限、装入、拒绝、轮转公平性、完成释放、空队列、全通关、轮换和双击防护。
- 组件测试覆盖三课解锁、内存格、就绪队列、CPU、历史、触控与辅助模式。
- 发布前运行全量门禁与公网懒加载资源验证。
