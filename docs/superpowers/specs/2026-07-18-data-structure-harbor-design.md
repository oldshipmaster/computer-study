# 数据结构装卸港设计

## 目标

把数组、链表、栈、队列、树和图从五门分散课程连接成一场约 8–10 分钟的港口调度游戏。孩子不是背定义，而是通过“从哪里取、沿什么走、谁先离开、怎样到达”亲手证明每种结构的规则。

## 解锁与入口

- 完成 `array-lockers`、`linked-treasure`、`stack-queue-dock`、`tree-library`、`graph-routes` 后解锁。
- 在统一游戏中心登记为第九种玩法，目标锚点 `data-structure-harbor`。
- 锁定卡显示 0–5 进度并打开下一门未完成的数据结构课程。

## 六座码头

1. **数组货柜排**：根据连续索引直接读取和更新指定货柜，强调位置与地址的固定对应。
2. **链表寻宝线**：不按画面距离猜测，而是从头节点沿 `next` 指针逐站到达目标。
3. **栈顶卸货塔**：货物只从顶部进出，按后进先出依次弹出。
4. **队首通行桥**：乘客从队尾加入、队首离开，按先进先出保持公平。
5. **树形图书馆**：从根节点按父子分支走向目标叶子，留下完整路径。
6. **图路线调度台**：在有环路线中选择相邻节点并找到不重复的最短可达路径。

## 互动规则

- 每一步显示结构快照：索引、地址、指针、栈顶/队首、父子层级或图节点。
- 正确动作写入右侧“结构证据单”；错误动作保留结构并提示应该观察的关系。
- 正确反馈停留到孩子主动继续；双击、未知动作和非当前阶段输入不会重复推进。
- 六关结束总结“直接索引、指针遍历、后进先出、先进先出、层级路径、图最短路”六种能力。
- 重玩轮换关卡顺序；不设倒计时、生命值、排行榜或随机惩罚。

## 无障碍、性能与隐私

- 控件至少 44px，当前节点同时使用文字、边框和 `aria-current`，不只依赖颜色。
- 移动端单栏，横向结构允许安全滚动；减少动态和强制颜色模式都有明确规则。
- 组件及样式按需加载。操作轨迹仅在当前页面内存存在，不写入本机进度或上传。

## 验证

- 纯引擎测试：六关目录、错误保留、证据推进、双击保护、通关稳定与轮换。
- 组件契约：五课解锁、六种结构标签、焦点交接、游戏中心登记、触控与响应式规则。
- 全量测试、类型、lint、依赖审计、Pages 构建、独立资源和线上关键文案验证。
