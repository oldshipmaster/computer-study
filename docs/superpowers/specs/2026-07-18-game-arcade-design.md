# 比特岛游戏中心设计

## 问题

课程地图已经拥有探险任务、闪击赛、Boss、逻辑电路、机器人代码、网络护航和 CPU 调度。继续把每个入口并列放在页头，会让七岁儿童面对过多导航选项，也难以理解哪些已经解锁、下一步学什么。

## 方案

- 在所有独立游戏之前增加“比特岛游戏中心”。
- 页头将多个游戏锚点收敛为一个“游戏中心”入口；具体玩法保留原锚点，从中心卡片直达。
- 每张卡显示图标、名称、核心操作、预计时长、解锁进度和状态。
- 已解锁卡片使用原生锚点链接；未解锁卡片提供下一门必修课按钮。
- 目录与 CSS 独立懒加载，避免增加首屏主包。

## 七类玩法与真实条件

1. 探险任务牌：始终开放。
2. 比特知识闪击赛：至少完成 3 门已注册课程。
3. 十三岛 Boss 战：至少完整完成任意一座岛的 5 门课；进度显示目前完成最多的一座岛。
4. 逻辑电路实验台：完成 `bits-and-data`、`boolean-logic`。
5. 机器人代码远征：完成 `instruction-order`、`grid-city-navigation`、`repeat-power`、`rainy-condition`。
6. 网络数据包护航：完成 `network-layers`、`routing-maze`、`reliable-transfer`。
7. CPU 时间片调度台：完成 `program-process`、`cpu-scheduling`、`memory-allocation`。

未知、重复课程 ID 不计入进度。Boss 下一课从当前最接近完成的岛中选择，其他游戏按固定必修顺序选择。

## 体验与验证

- 卡片不引入签到、每日奖励、排行榜或新持久化数据。
- 触控目标至少 44px，状态不只依赖颜色，支持移动端、减少动态和强制颜色。
- 引擎测试覆盖目录唯一性、全部解锁条件、Boss 最大岛进度、下一课、未知/重复 ID。
- 组件测试覆盖卡片、进度、直达入口、懒加载和页头收敛。
