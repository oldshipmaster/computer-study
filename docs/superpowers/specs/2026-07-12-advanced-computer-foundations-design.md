# 比特岛进阶计算机基础课程设计

## 1. 目标

在现有“比特岛大冒险”九岛四十五课基础上，新增四座专题岛和二十节完整互动课，把数据结构与算法、计算机组成原理、计算机网络、操作系统系统性融入小学二年级可理解的学习路线。完成后课程体系为十三岛六十五课，旧进度无需迁移或重置。

## 2. 学习者与课程约束

- 学习者：小学二年级，约七岁。
- 学习方式：孩子自主探索，家长偶尔协助。
- 单课时长：八至十分钟。
- 表达方式：先用可视化游戏形成模型，再引入准确术语。
- 不要求记忆复杂公式、书写正式代码或操作真实系统设置。
- 每课包含故事引入、概念讲解、引导练习、独立挑战、完成反馈和课后复盘。
- 错误尝试不扣除既有进度，反馈必须解释原因和下一步观察点。

## 3. 课程架构

### 3.1 第十岛：数据结构群岛

1. **数组储物柜**：按编号存取物品，理解顺序存放、索引、读取和更新。
2. **链表寻宝线**：沿“下一站”指针遍历，体验在中间插入、删除和断链修复。
3. **栈与队列码头**：用盘子堆和候船队比较后进先出与先进先出。
4. **树形图书馆**：从根节点沿分类分支查找，理解父节点、子节点、叶节点和层级。
5. **图结构航线网**：在节点和边组成的航线中判断相邻、可达、路径和环。

### 3.2 第十一岛：算法竞技场

1. **顺序查找侦察**：逐项检查目标并记录比较次数。
2. **二分查找猜数**：在有序数据中比较中间项并每次排除一半。
3. **排序机器人**：通过相邻比较与交换完成冒泡排序，观察每一轮的变化。
4. **拆分任务工坊**：把复杂任务拆成小任务，识别依赖并按可执行顺序组合。
5. **算法效率赛**：在不同数据量下比较顺序查找和二分查找的操作次数，不引入大 O 公式。

### 3.3 第十二岛：操作系统控制塔

1. **程序变成进程**：启动、运行、暂停、恢复和结束虚拟程序，区分程序文件与运行中的进程。
2. **CPU 调度转盘**：让多个任务轮流获得时间片，理解就绪、运行、等待和公平调度。
3. **内存房间管理**：为进程分配与释放有限内存，识别空间不足和结束任务后的回收。
4. **文件系统目录树**：沿路径查找目录与文件，理解命名、层级、权限和回收站。
5. **设备协调中心**：让操作系统依次处理键盘、屏幕、打印机和程序请求，理解设备驱动的桥梁作用。

### 3.4 第十三岛：系统与网络深海站

1. **指令执行流水线**：追踪取指、译码、执行和写回四步，理解 CPU 按时钟重复工作。
2. **缓存快递站**：比较从缓存、内存和存储取得数据的等待成本，理解局部性和命中。
3. **网络分层信封**：将消息依次装入应用、传输、网络和链路层信息，再在终点逐层拆开。
4. **路由选择迷宫**：根据连接状态与路线代价选择下一跳，并在链路中断后重新规划。
5. **可靠传输救援**：用编号、确认、超时、重传和排序恢复丢失或乱序的数据块。

## 4. 难度梯度与交叉路线

- 六十五课仍按十三个领域岛组织，但推荐学习路线分五轮交叉推进，不要求连续学完一座岛。
- 新四岛的前两课为具体操作模型，中间两课引入关系与状态，最后一课要求比较、解释或综合应用。
- 新课会主动关联旧课：数组关联文件表格，栈与队列关联程序顺序，进程关联桌面窗口，缓存关联 CPU/内存/存储，网络分层与可靠传输关联数据包旅行。
- 完成旧四十五课的用户从新的第一个未完成课程继续，不更改既有完成记录、徽章、自评或续课状态。

## 5. 互动结构

每节新课使用统一六阶段课程外壳：

1. 场景和学习目标。
2. 一个核心模型的可视化演示。
3. 引导操作并即时反馈。
4. 预测下一步结果。
5. 纠错或比较任务。
6. 独立挑战与完成奖励。

纯逻辑模型与 React 界面分离。数组、链表、查找、排序、调度、内存分配、缓存、路由和可靠传输分别建立小型、确定性的状态转换模块；互动组件只负责展示状态和提交动作。

## 6. 课程数据与产品接入

- 课程目录从九岛四十五课扩展为十三岛六十五课，每座岛五课。
- 新课程使用现有课程注册表、完成徽章、续课阶段、反思自评、知识图鉴、课程搜索和家长区。
- 为二十节新课分别增加三个学习目标、一个家长讨论问题、一个核心词典术语和一道概念配对题。
- 为四座新岛增加至少八道跨领域情境题，每岛两道。
- 课程统计、家庭学习计划、结业证书、岛屿印章和进度百分比全部从课程目录动态派生。

## 7. 进度兼容

- 继续使用稳定的课程字符串 ID；原四十五个 ID 不改变。
- 读取旧进度时保留所有已知旧课程、徽章、设置和自评。
- 旧备份文件可以导入新版本；新增课程在旧备份中自然处于未完成状态。
- 新备份仍使用字段白名单，不包含姓名、自由文本、答题过程或真实账号。
- 离线缓存升级不得先切换到引用未缓存资源的新页面。

## 8. 安全与隐私

- 所有文件、内存、进程、路由和传输实验只在组件内存中模拟。
- 不调用 `fetch`、XHR、WebSocket、EventSource、`sendBeacon` 或真实导航 API。
- 不读取真实文件、进程、网络配置、浏览记录或系统信息。
- 不要求儿童输入姓名、电话、学校、住址、账号或密码。
- 真实设备、路由器与操作系统设置只允许观察，不引导儿童独自修改。

## 9. 无障碍与响应式要求

- 所有操作同时支持鼠标、触摸和键盘；不得依赖拖拽作为唯一操作方式。
- 交互目标至少四十四像素，核心课程文字在移动端至少十六像素。
- 新阶段将焦点移动到课程一级标题，课程退出后恢复到原课程卡。
- 状态变化通过可见文字和礼貌型实时区域表达，不只依赖颜色、动画或声音。
- 支持高对比度、减少动画、关闭声音和三百九十像素宽屏幕。
- 三百二十像素宽屏幕不得产生水平溢出。

## 10. 测试与验收

### 10.1 领域逻辑

- 每个新增状态转换函数先写失败测试，再实现最小逻辑。
- 覆盖正常路径、无效输入、边界状态、不变性和重复激活。
- 查找和效率课程必须验证比较次数；调度与传输必须验证确定性和不丢状态。

### 10.2 课程完整性

- 十三座岛各拥有且只拥有五个唯一课程 ID。
- 六十五课全部标记可玩，并在注册表中存在完整定义。
- 每课时长在八至十分钟，具有三项目标和一个家长问题。
- 六十五个核心词典条目与课程一一对应。
- 跨领域情境题覆盖全部十三岛。

### 10.3 产品质量

- 全量单元测试、服务端渲染测试、TypeScript、ESLint 和依赖审计通过。
- GitHub Pages 静态产物保持 CSP、隐私、离线、图标和性能预算。
- 在桌面、三百九十像素和三百二十像素视口验证二十节新课无溢出、无小触控目标和无运行时错误。
- 验证键盘焦点、高对比度、减少动画、离线重访和旧进度恢复。
- GitHub Actions 成功部署，线上首页与静态资源和本地产物一致。

## 11. 完成定义

只有在二十节新课均能从地图进入、完成、重玩和恢复，四个领域的明确知识目标都有直接互动证据，所有配套内容与六十五课目录对齐，旧进度保持兼容，并且本地与线上全部验收通过后，才视为本次扩展完成。
