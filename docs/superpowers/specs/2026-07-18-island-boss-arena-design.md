# 十三岛 Boss 战设计

## 目标

让孩子在完成一整座岛的五节课后，通过一场 4–6 分钟的综合任务把同一领域的知识连接起来。Boss 战不是考试排名，也不是单题换皮：每场都依次要求找证据、排操作顺序、解释原理。

## 核心循环

1. 完成某座岛的五节课程，解锁该岛 Boss。
2. 从 Boss 雷达选择已解锁岛屿，阅读虚构故障或任务简报。
3. **证据扫描**：从四条观察中选出两条真正有用的证据，再提交判断。
4. **行动编队**：从四个候选行动中挑出三个，并按安全、有效的顺序排入队列；可以撤回重排。
5. **原理核心**：从三个解释中选择能说明“为什么”的答案。
6. 三阶段完成后点亮该岛 Boss 核心；可随时重玩，不消耗探险能量。

## 反馈与失败

- 没有倒计时、生命值、淘汰或负分。
- 提交错误时保留当前操作，说明需要重新观察的维度，不直接公布完整答案。
- 每一阶段完成后显示本阶段学到的迁移规则，再进入下一阶段。
- 重复点击或双击不能越过阶段；完成状态不可被后续输入改变。

## 内容范围

十三座岛各一场独立 Boss：基础操作、文件管理、编程思维、数字安全、硬件、网络、数字创作、协作与 AI、编程构件、数据结构、算法、操作系统、系统与网络。每场数据包含：简报、四条证据、正确证据 ID、四个候选行动、三个正确行动的顺序、三个原理解释与迁移总结。

所有场景只在页面内模拟，不打开真实文件、系统设置或网络连接。网络与账号场景使用虚构设备和保留示例域名，不让儿童输入姓名、账号、密码或自由文本。

## 解锁与本机记录

- 解锁条件严格来自岛屿的五个课程 ID；重复或未知完成记录不影响判断。
- 新增 `completedBossIds`，最多保存十三个已知 Boss ID。
- 只保存完成的 Boss ID，不保存孩子选过的证据、操作队列、错误次数或回答历史。
- 备份、恢复与目录清洗继续使用字段白名单；旧进度自动迁移为空数组。

## 界面

- 地图任务牌、知识闪击赛之后增加 `#island-boss-arena`。
- 雷达卡片清楚区分锁定、可挑战、已点亮；锁定卡展示 `已完成 n / 5 课`，并能打开该岛下一节未完成课程。
- 战斗区使用深海控制台，但保持高对比、44px 触控目标、原生按钮和状态播报。
- 手机端单列；键盘操作可完整完成；阶段变化移动焦点到新标题；减少动画和强制颜色模式保留状态差异。

## 明确不做

- 不做排行榜、分享、签到、连续天数、随机抽奖或内购式货币。
- 不把 Boss 胜利计入课程完成数或探险能量，避免重复奖励含义。
- 不增加账号、云同步、远程题库或网络请求。
