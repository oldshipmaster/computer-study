# 网络数据包护航设计

## 目标

把网络分层、最短路由、分组编号、丢包、确认、重传、按序组装和反向拆层串成一套端到端互动游戏。孩子不回答定义题，而是亲手把一条虚构学习消息安全送过虚拟网络。

## 解锁与入口

- 完成 `network-layers`、`routing-maze`、`reliable-transfer` 三门课程后解锁。
- 入口位于机器人代码远征之后、五次学习计划之前。
- 未解锁时显示 0–3 门进度，并可直接打开下一门必修课。
- 组件和样式独立懒加载。

## 六次护航

每次任务使用不同的虚构消息、3–5 个编号分组、三条候选路线、一个预设丢失分组和一种乱序到达方式。路线名称、节点和代价全部是本地模拟，不访问真实网络。

每次都经过以下阶段：

1. **逐层打包**：依次加入应用层、传输层、网络层、链路层。错误层不会加入，并解释为什么内部信息要先包装。
2. **选择路线**：比较每条路线的总成本，选择代价最低的一条。不是比较节点数量。
3. **发射分组**：一次模拟发送全部编号分组，保留到达与丢失证据。
4. **精确重传**：根据缺少的编号，只重传丢失分组；错选不会清空证据。
5. **接收排序**：用上移、下移把乱序分组排成编号顺序，再确认组装。
6. **反向拆层**：依次移除链路层、网络层、传输层、应用层，最后恢复消息。

## 状态与反馈

- 所有动作由纯函数状态机约束，非法阶段、无效 ID 和双击第二次激活均忽略。
- 每一步保留已完成证据；错误只提示当前缺口，不回退整关。
- 完成六次后可轮换任务顺序重玩。
- 游戏不写入学习进度、答案历史或备份，只存在当前页面内存。

## 儿童体验与安全

- 无倒计时、生命值、随机抽奖、排行榜和真实网络请求。
- 包裹颜色始终配有层名；路线同时显示节点和数字代价；分组同时显示编号与内容片段。
- 所有按钮至少 44px，支持键盘、屏幕阅读器、移动端、减少动态和强制颜色。
- 一次护航约 1–2 分钟，六次形成约 8–10 分钟完整小课。

## 验证

- 引擎测试覆盖六套任务、层顺序、最短路线、丢包证据、精确重传、乱序调整、组装、反向拆层、轮换、非法动作与双击防护。
- 组件测试覆盖解锁、阶段视觉、语义、触控尺寸、焦点和地图接入。
- 发布前完成全量测试、类型、lint、依赖审计、Pages 资源预算和公网懒加载资源复核。
