# 机器人代码远征设计

## 目标

为已完成机器人工作坊核心课程的孩子提供一套可反复玩的“编程后执行”游戏。玩法必须和知识闪击赛、Boss 战、逻辑电路实验台不同：孩子编辑指令队列，通过逐步执行观察程序状态，并用循环与条件压缩后期关卡。

## 解锁和入口

- 入口位于逻辑电路实验台之后、五次学习计划之前。
- 完成 `instruction-order`、`grid-city-navigation`、`repeat-power`、`rainy-condition` 后解锁。
- 未解锁时显示完成数量，并提供下一门必修课按钮。
- 组件懒加载，样式单独拆包，不增加首屏主包压力。

## 六关结构

每关是 5 × 5 的虚拟方格地图，包含起点方向、能量点、终点、障碍和指令上限。所有移动只发生在本地虚拟地图。

1. 直线收能量：练习从上到下执行。
2. 转弯到终点：组合前进和转向。
3. 重复压缩：必须使用“重复前进 2 格”。
4. 障碍传感器：必须使用“如果前方挡住就右转”。
5. 条件与循环组合：先移动到观察位置，再根据障碍转向。
6. 综合远征：用顺序、循环、条件和方向完成较长路线。

可用指令：前进一格、左转、右转、重复前进两格、如果前方挡住就右转。每关只开放已经需要的指令，减少七岁儿童的选择负担。

## 交互状态

- `building`：添加、上移、下移、删除或清空指令；不能超过关卡上限。
- `ready`：至少一条指令后可运行。
- `tracing`：一次只执行一个展开后的原子步骤，当前指令高亮，地图同步更新。
- `failed`：撞墙、越界、未收齐能量或未到终点时显示可修复证据，原程序保留。
- `solved`：到达终点并收齐能量，可进入下一关。
- `complete`：六关通关，可轮换关卡顺序重玩。

重复指令展开为两个连续的前进步骤；条件指令在运行当下读取前方格子，因此同一条程序会根据现场状态产生不同结果。执行记录只保存在当前组件内存，不写入本机进度、备份或网络。

## 儿童体验与安全

- 无倒计时、生命值、签到、随机奖励、排行榜和失败惩罚。
- 失败显示“在哪一步、看到什么、下一步可以改什么”，不直接给出完整答案。
- 所有按钮至少 44px；地图同时用图形、文字和位置状态表达，不依赖颜色。
- 支持键盘、屏幕阅读器、减少动态和强制颜色模式。
- 完成一次程序约 1–2 分钟，六关形成约 8–10 分钟完整小课。

## 验证

- 引擎测试覆盖地图合法性、指令上限、重复展开、条件运行时判断、碰撞、能量收集、成功与失败、重玩轮换、非法/双击输入。
- 组件契约测试覆盖解锁、程序编辑、逐步执行、地图语义、焦点移动、响应式和辅助模式。
- 完成全量测试、类型、lint、依赖审计、Pages 构建和公网懒加载资源复核。
