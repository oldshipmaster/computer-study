# 逻辑电路实验台设计

## 目标

增加一种与问答、课程卡片、Boss 选择题明显不同的可重复操作游戏。孩子需要亲手把逻辑门放进电路，再让所有输入组合通过测试；不是猜一次答案，而是用完整真值表验证规则是否对每种情况都成立。

## 解锁

- 同时完成 `bits-and-data` 与 `boolean-logic` 后开放。
- 锁定态显示 0/2、1/2 或 2/2，并提供下一门必修课程按钮。
- 不改变课程完成数、探险能量或 Boss 进度。

## 挑战模式

- 六块电路板：AND、OR、NOT、XOR、两层 `(A AND B) OR C`、两层 `NOT(A OR B)`。
- 每块板显示输入端、一个或两个空逻辑门槽、输出灯和目标真值表。
- 孩子为每个槽选择逻辑门，再按“运行全部输入”。
- 引擎自动跑遍 1–3 个输入的所有组合，逐行显示实际输出与目标输出。
- 任意一行不符时保留门槽选择，标出失败行，允许继续修改；全部吻合才点亮本关核心。
- 一次通关后可以轮换顺序重玩，也可以进入自由实验。

## 自由实验

- A、B 两个开关可随意切换。
- 可替换 AND、OR、XOR、NOT 逻辑门，输出灯立即变化。
- 同时用中文规则和符号显示当前关系，避免只靠颜色传达。
- 自由实验没有分数、对错或完成要求。

## 状态与隐私

- 游戏状态只存在于当前 React 组件内，不写入本机进度。
- 不保存电路选择、测试失败、操作次数或自由实验状态。
- 不发网络请求，不访问真实设备或系统接口。

## 可访问性与性能

- 所有门槽、开关和操作都使用原生按钮，最小触控高度 44px。
- 真值表使用语义表格，失败行还显示文字“需调整”，不只变色。
- 新组件与专属 CSS 延迟加载；锁定提示可在首屏服务端输出。
- 阶段切换移动焦点；状态通过 `role=status` 播报。
- 手机单列、减少动画禁用输出灯脉冲、强制颜色保留选中与失败边框。

## 明确不做

- 不做倒计时、排行榜、生命值、签到或随机奖励。
- 不要求孩子输入代码、姓名或自由文本。
- 不把逻辑门模拟描述为真实电气工程操作；这是抽象布尔规则实验。
