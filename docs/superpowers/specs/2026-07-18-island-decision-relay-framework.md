# 岛屿六关接力框架设计

## 目标

为仍缺少综合实战的五座岛快速增加完整 8–10 分钟游戏，同时统一安全、错误恢复、焦点、触控和重玩标准。框架只负责互动状态与通用画布，各岛保留独立任务数据、标题、颜色和视觉符号。

## 数据模型

- 每个游戏由六关组成，每关两个判断步骤。
- 每步包含 2–4 张固定虚构信号卡、三个动作、唯一正确动作、解释、证据和错误提示。
- 每关标明对应课程和能力，完成页展示六项能力。

## 状态规则

- `active → step-solved → mission-solved → complete`。
- 错误选择只更新反馈，不删除已有证据或改变画布。
- 未知动作、阶段外动作和双击第二次激活全部忽略。
- 重玩按轮换值改变关卡顺序，不丢失或重复关卡。

## 组件规则

- 由包装组件提供课程门槛、主题、文案、图标和任务。
- 通用组件负责锁定页、入口、进行中、证据板和完成页。
- 不提供输入框、上传、真实网络、倒计时、排行榜或行为存储。
- 支持 44px 触控、键盘焦点、状态播报、移动端单列、减少动画和强制颜色。
