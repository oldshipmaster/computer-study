# 比特知识闪击赛设计

## 目标

在 65 节完整课程之外，增加一个可反复游玩、每局约 2–4 分钟的短挑战。它从孩子已经学过的内容中动态组题，让课程之间也有“马上能玩一局”的入口，同时用连击、护盾和本机最佳成绩制造游戏反馈，不用倒计时、淘汰或连续签到制造压力。

## 路线比较

### 路线 A：给每课增加困难模式

变化最贴近课程实验，但需要分别改造 65 套交互，规则和质量难以统一，也会显著扩大维护面，本轮不采用。

### 路线 B：五题知识闪击赛（采用）

复用现有 65 道概念题与 26 道情境题，按已完成课程动态抽取一局，形成跨课程的短局循环。它能快速产生组合变化，也能把答错的概念导回原课程。

### 路线 C：扩大比比换装

收藏反馈明显，但单独扩展会让奖励系统比学习互动更突出。保留现有五级装备，不在本轮做商店或货币。

## 核心玩法

地图在探险任务牌之后增加“比特知识闪击赛”。每局最多五题，题目来自孩子已经完成课程解锁的内容：

- 概念题来自现有 `TERM_MATCH_QUESTIONS`，要求根据解释选出正确术语。
- 情境题来自现有 `REVIEW_QUESTIONS`，要求在真实电脑场景中选出最合理行动。
- 组题优先按“概念、情境、概念、情境、概念”交替；某一类型不足时由另一类型补齐。
- 同一局不重复题目，同一完成进度与同一轮换编号生成相同题组。
- 点击“再来一局”增加轮换编号，尽可能换一组题；题库不足五题时显示真实题数，不复制题目凑数。

少于三道已解锁题时，挑战台保持锁定，明确显示“再完成课程来解锁”，并提供打开下一课的按钮。它不以灰色假按钮诱导点击。

## 计分与反馈

每局开始有三格“比比护盾”，护盾只是反馈，不会让孩子出局：

- 答对：基础 100 分；当前连续答对每多一题，本题再加 25 分。
- 答错：本题 0 分、连击归零、护盾减少一格，但会显示解释并允许继续下一题。
- 护盾归零后仍能完成全局，文案改为“修复模式”，避免失败终止学习。
- 每题只能作答一次；回答后先阅读反馈，再点击“下一题”。双击或重复激活不能跳过题目。
- 局末显示总分、答对数、最高连击与本机最佳分。
- 如果有答错题，提供“回到对应课程加练”按钮，打开本局第一道答错题关联的课程。
- 满分时显示“全连击”，但不给可无限刷取的探险能量。

最高理论分为 750：连续答对五题分别得到 100、125、150、175、200 分。分数只用于本机自我比较，不提供排行榜、排名或与其他孩子比较。

## 状态模型

`lib/knowledge-sprint.ts` 提供纯函数：

- `buildKnowledgeSprintDeck(completedCourseIds, rotation, limit = 5)`：构建确定、交替、无重复题组。
- `createKnowledgeSprintState(questionCount)`：创建本局状态。
- `answerKnowledgeSprint(state, optionIndex, deck, activationDetail)`：处理一次作答，进入反馈阶段。
- `advanceKnowledgeSprint(state, deck, activationDetail)`：从反馈进入下一题或完成页。

状态包含 `index`、`phase`、`score`、`correct`、`streak`、`bestStreak`、`shields`、`feedback` 和 `missedCourseIds`。作答函数不修改题组或旧状态。

`components/KnowledgeSprint.tsx` 只负责渲染、焦点、轮换和向上报告成绩。完成回调每局只触发一次。

## 本机记录与兼容性

现有 `bit-island-progress-v1` 增加：

```ts
knowledgeSprint: {
  bestScore: number;
  runsPlayed: number;
}
```

- `bestScore` 只接受 0–750 的整数。
- `runsPlayed` 只接受 0–10,000 的整数并在记录时封顶。
- 旧记录缺字段时自动补 `{ bestScore: 0, runsPlayed: 0 }`。
- `recordKnowledgeSprint(progress, score)` 只接受合法分数，更新最高分并让局数加一。
- 字段进入现有目录清洗、JSON 白名单备份、恢复和重置；重置会清零成绩但保留声音与减少动画设置。
- 不存每题答案、作答历史、孩子姓名或自由文本。

## 界面与可访问性

- 挑战区在任务牌后、五次计划前，首屏向下即可到达；顶部快捷航线增加“闪击赛”。
- 题目标题在每次换题后获得焦点，完成页标题获得焦点。
- 所有答案和下一题都使用真实按钮，触控高度至少 44px。
- 状态反馈使用 `role="status"`；护盾同时提供文字“剩余 N / 3”，不只依赖图标或颜色。
- 题目进度使用原生 `progress`，并显示“第 N / M 题”。
- 颜色不作为正确或错误的唯一信息；答案后明确写“答对了”或“这一题没有答对”。
- 680px 以下改为单列答案和紧凑状态栏。
- 减少动画开启时禁用连击脉冲、护盾摇动和卡片位移。
- 不自动播放声音、不使用倒计时。

## 边界与错误处理

- 未完成课程：锁定挑战，显示下一课入口。
- 只有 1–2 道题：继续锁定，避免把同题复制成一局。
- 题库恰好 3–4 道：允许短局，进度与满分按实际数量计算。
- 所有题都已解锁：每次轮换从完整题库生成不同的五题组合。
- 无效选项、双击、完成后继续作答：返回原状态。
- 无效轮换值：按 0 处理。
- 存储不可用：本局仍可玩，本机最佳分沿用当前内存状态；继续显示现有存储警告。

## 测试与验收

1. 先为题组生成写失败测试，覆盖锁定、交替、无重复、确定性、轮换和完整题库。
2. 为计分状态机写失败测试，覆盖连击加分、答错护盾、反馈阶段、双击、继续、完成和错题课程。
3. 为进度字段写失败测试，覆盖旧记录迁移、范围清洗、记录最高分、局数封顶、重置和备份恢复。
4. 为组件源码与服务器渲染写失败测试，覆盖锁定态、题目区、原生进度、状态文字、焦点和课程加练按钮。
5. 检查触控、移动布局、减少动画与强制颜色模式。
6. 运行生产构建、393 项现有测试加新增测试、类型、代码规范、依赖审计和 GitHub Pages 输出门禁。
7. 推送后确认 GitHub Actions build/deploy 成功，公开页面 HTTP 200 且线上资源包含“比特知识闪击赛”。

## 本轮不做

- 倒计时、限时奖励、生命耗尽后强制结束。
- 排行榜、账号、云同步或与他人比较。
- 闪击赛探险能量奖励，避免重复刷分。
- 新增外部题库、网络请求或收集答案历史。
- 改写 65 节课程内部实验。
