# Hardware Lab and Network Bay Expansion

**Goal:** Close the two remaining elementary-computing foundation gaps with ten additional 8–10 minute interactive lessons, bringing Bit Island from 20 to 30 complete courses.

## Hardware Lab

21. `input-process-output`: trace input → processing → output through familiar devices.
22. `cpu-memory-storage`: distinguish CPU work, short-term memory, and long-term storage.
23. `bits-and-data`: build small binary patterns and see how numbers, colors, and symbols become data.
24. `hardware-software`: match physical components with programs and explain how they cooperate.
25. `troubleshoot-machine`: use observe → check basics → change one thing → retest for common fictional device problems.

## Network Bay

26. `network-journey`: send a message through device, router, network, server, and back.
27. `web-address`: read a fictional web address, distinguish site identity from path, and recognize lookalikes.
28. `search-and-links`: form useful search terms, compare result clues, manage links and tabs safely.
29. `downloads-and-cloud`: distinguish local, downloaded, uploaded, synced, and shared copies without performing real transfers.
30. `network-troubleshooting`: diagnose offline, weak connection, server unavailable, and wrong-address cases without changing real settings.

## Guardrails

- Every network, URL, file transfer, and device is a closed simulation with fictional `.example` addresses.
- No real browsing, downloading, uploading, sharing, telemetry, or personal information.
- Pure models are deterministic and test boundaries, ordering, duplication, privacy, and safe troubleshooting.
- Enable each catalog entry only with its finished component, registry definition, focused tests, full build, Pages build, and lint pass.
