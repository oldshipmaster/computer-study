# 比特岛大冒险

「比特岛大冒险」是面向小学二年级儿童的电脑端互动启蒙课程。孩子跟随机器人向导比比探索十三座岛屿，在短时、可操作的冒险中交替学习电脑基础、编程、数据结构、算法、操作系统、计算机组成和网络知识。

## 当前课程范围

- 展示十三座岛屿、65 节课程的完整学习路线、目标与完成状态。
- 提供课程地图、本机进度、徽章、声音与动画设置，以及受保护的家长区。
- **十三座岛屿的 65 节课程均可完整游玩**：覆盖电脑操作、文件管理、编程思维、数字安全、数据结构、算法、操作系统、计算机组成、互联网、数字创作、在线协作与 AI 素养。
- 每节课采用约 8–10 分钟的故事、讲解、引导练习、独立挑战和奖励节奏；完成课程后可随时从地图重玩。
- 所有课程都通过统一的课程注册表接入地图、续课进度、完成徽章和通用完成页；只有已经实现并通过测试的课程才会标记为可玩。
- 课程地图还提供五次滚动学习计划、知识图鉴、65 词儿童电脑词典、65 道逐课概念配对题、26 道十三岛情境题与孩子自主加练清单。
- 地图新增“比比的探险任务牌”：把新课、低频重玩和孩子自己标记的薄弱课程组合成最多三张任务；每课首次完整通关获得 10 点探险能量，第 2、3 次完整重玩各获得 3 点，并自动点亮五级比比装备，不使用签到、抽奖或排行榜。
- 新增统一的“比特岛游戏中心”：二十张游戏卡集中展示探险任务、闪击赛、Boss 战以及十三座岛的专属综合玩法，并提供完成 65 课后开放的“十三岛冠军联赛”；孩子可以直接进入已解锁游戏，未解锁卡片会显示学习进度并打开下一门前置课程。
- 游戏中心会根据“我有 10 / 20 / 30 分钟”从已解锁玩法中跨主题安排 1 / 2 / 3 局，支持按标题或玩法说明本地搜索，并提供“综合挑战、编程与逻辑、电脑与网络、安全与文件”主题筛选、“入门探险、进阶挑战、大师联赛”阶段筛选和只看已解锁；每张卡会标明阶段，时间选择、搜索词、推荐轮换与筛选只在当前页面发生，不形成行为画像。
- 完成足够课程后可进入“比特知识闪击赛”：每局从已学内容中交替抽取五道概念题与情境题，用连击分和三枚护盾提供即时反馈；答错不会出局，战报可直接回到对应课程加练，也能换一组题继续挑战。
- 每完成一整座岛的 5 节课程，就会解锁该岛的综合 Boss 战。十三场任务分别要求孩子从虚构场景中找出两条关键证据、把三个安全行动排成有效顺序，再选出能解释因果关系的核心原理；没有倒计时或淘汰，错误选择会保留供孩子继续调整。
- 完成“比特积木”和“真假开关”后可解锁“逻辑电路实验台”：孩子亲手选择 AND、OR、XOR、NOT 逻辑门，用完整真值表测试六块逐步加深的电路板；通关后可以轮换重玩，也可以在无目标、无对错的自由实验中切换输入并即时观察输出。
- 完成机器人车间顺序、坐标、循环、条件和调试五门课程后可解锁“机器人代码远征”：孩子在六张 5×5 地图上编辑指令队列，再逐步执行并观察机器人位置、朝向和能量状态；后半程要使用重复块与运行时条件压缩程序，失败会保留原程序供继续调试。
- 完成网络分层、路由迷宫和可靠传输后可解锁“网络数据包护航”：孩子把虚构消息由内向外打包，比较三条路线的总代价，根据到达编号精确重传丢失分组，整理乱序内容，再由外向内拆层恢复消息；六次任务把网络传输全过程连成约 8–10 分钟的互动小课。
- 完成程序进程、CPU 调度和内存分配后可解锁“CPU 时间片调度台”：孩子在 4–8 格有限内存中装入虚构任务，让 CPU 每次只运行就绪队首的一个时间片；未完成任务轮转到队尾，完成任务自动释放内存，六个班次逐步形成进程、调度与资源释放的系统视角。
- 完成算法岛五门课程后可解锁“算法竞技场”：孩子在六场接力中按顺序执行线性搜索、利用中点缩小二分范围、交换冒泡排序的相邻数字、按依赖解锁任务，再依据资料是否有序和相同输入下的步骤数选择方法；错误操作保留当前证据供继续判断。
- 完成数据结构岛五门课程后可解锁“数据结构装卸港”：孩子在六座码头中直接按数组索引取货、沿链表指针寻宝、分别按后进先出和先进先出调度栈与队列、沿树的父子分支找叶子，再在有环图中比较边数寻找最短路线。
- 完成安全灯塔五门课程后可解锁“数字安全侦探局”：孩子只使用虚构线索侦破隐私包裹、口令保险箱、弹窗迷雾、健康值守、共享设备收尾和灯塔总案；不输入真实资料、不打开链接，风险响应始终强调停止操作并告诉可信大人。
- 完成硬件实验岛五门课程后可解锁“虚拟电脑装配厂”：孩子在六个屏幕内工位连接输入、CPU 处理、工作内存、长期存储与输出，用位值组合比特、为任务配齐软硬件，并按“观察、只改一处、复测”的安全流程排错；不要求拆机或接触真实电源。
- 完成文件森林五门课程后可解锁“文件森林救援队”：孩子只操作虚拟文件卡，在六次救援中沿完整路径寻路、保留扩展名清楚命名、区分移动原件和复制副本、依据类型分类，并从回收站恢复后核对名称与路径；不读取或修改真实文件。
- 完成创作工坊五门课程后可解锁“创作工坊项目赛”：孩子用固定虚构素材完成像素图纸、文档编辑、幻灯片导演、素材版权、数据表设计和综合发布六个项目，每个正确决定都会留下可讲述的设计理由；不上传文件或输入个人资料。
- 完成未来协作站五门课程后可解锁“AI 核验研究站”：孩子在六项离线虚构任务中核对邮件、安排协作权限、构造不含个人资料的 AI 提问、拆分并交叉核验 AI 主张、规划项目依赖，最后区分人类决定、工具建议和资料证据。
- 完成代码星港五门课程后可解锁“迷你游戏导演”：孩子把事件接到处理程序，用变量保存分数和能量，用函数复用工具，用布尔逻辑设置规则，再按稳定步骤复现、修复和复测问题，并依据试玩证据调整公平难度。
- 完成启航港五门课程后可解锁“电脑驾驶执照”：孩子在六关虚拟操作中练习方向键与确认、单击和拖放、大小写与拼音组合、窗口焦点与最小化、程序身份与正常退出，最后按状态依赖完成综合驾驶考试。
- 完成网络海湾五门课程后可解锁“网络航海训练营”：孩子沿固定模拟路线理解设备、路由器与服务站，拆读网址结构，依据关键词、摘要和来源判断搜索结果，区分上传下载方向，并用“先观察、一次一变、复测、求助”排查故障。
- 完成操作系统峡谷五门课程后可解锁“操作系统任务指挥部”：孩子观察程序如何成为进程，让就绪任务轮流使用 CPU，比较有限内存容量，沿完整文件路径定位内容，管理设备请求队列，最后结合多个组件状态解释系统瓶颈。
- 完成系统深海五门课程后可解锁“系统深海总控台”：孩子跟踪取指、译码、执行和程序计数器变化，判断缓存命中与未命中，按层封装和拆封消息，依据路线代价与故障改路，再利用编号和确认信息精确重传缺失分组。
- 完成全部十三座岛的 65 课后可解锁“十三岛冠军联赛”：孩子在 13 个短回合中各解决一座岛的核心证据决策，覆盖操作、文件、编程、安全、硬件、网络、创作、AI、数据结构、算法、操作系统与系统原理，并可轮换顺序重赛。
- 每课结束可选择“我会讲了 / 我想再练 / 请大人帮忙”，选择只保存在本机，并在家长区形成可直接打开的复习建议。
- GitHub Pages 版本支持安装和离线重访；首次联网打开后会预缓存当前页面、脚本、样式、清单和应用图标。

## 隐私

网站无需登录，不收集儿童的姓名、照片、联系方式或其他个人信息。学习进度、徽章、每课最多三次的有效游玩次数、三选一自评、闪击赛本机最佳分与完成局数、已点亮的 Boss ID，以及偏好设置只保存在当前浏览器的本机存储中，不会上传到服务器；家长可以随时在家长区重置、导出或恢复这些数据。游戏中心的主题筛选与推荐轮换不写入存储。闪击赛不保存答案历史，Boss 战不保存证据选择、行动队列、回答或错误次数；逻辑电路实验台也不保存门槽选择、真值表结果或失败次数。机器人代码远征的程序队列与地图轨迹、网络数据包护航的打包与重传操作、CPU 调度台的任务队列与时间片历史、算法竞技场的选择与证据轨迹、数据结构装卸港的节点操作与证据单、数字安全侦探局的选择与案件证据、虚拟电脑装配厂的部件选择与检测单、文件森林救援队的虚拟文件选择与救援记录、创作工坊项目赛的设计选择与作品集、AI 核验研究站的示例选择与核验证据、迷你游戏导演的虚拟状态与导演记录、电脑驾驶执照的虚拟操作与驾驶记录、网络航海训练营的固定模拟状态与航海证据、操作系统任务指挥部的虚拟资源状态与调度记录、系统深海总控台的固定指令、路线、分组与系统日志，以及十三岛冠军联赛的虚拟回合与冠军证据册，都只存在于当前页面内存，不写入进度或备份。备份导出采用字段白名单，不包含自由文本、姓名、账号或答题内容。GitHub Pages 版本使用内容安全策略禁止连接外部服务和加载第三方脚本。若浏览器无法使用本机存储，课程仍可继续，但本次进度不会持久保存。

## 本地运行

需要 Node.js `>=22.13.0`。从仓库检出后按锁文件安装依赖：

```bash
npm ci
```

常用命令：

```bash
npm run dev
npm test
npm run lint
npm run typecheck
npm run audit:prod
npm audit
npm run test:pages
```

- `npm run dev`：启动本地开发服务器。
- `npm test`：执行生产构建并验证渲染结果。
- `npm run lint`：检查代码质量与格式规则。
- `npm run typecheck`：严格检查课程、进度与发布代码的 TypeScript 类型。
- `npm run audit:prod`：阻止带有中高危运行时依赖的版本进入发布流程。
- `npm audit`：同时检查本地构建工具与运行时依赖。
- `npm run test:pages`：构建并验证部署在 `/computer-study/` 下的 GitHub Pages 静态版本。
